# Selection Sort - 選択ソート

## 概要
選択ソートは、未ソート部分から最小値（または最大値）を見つけて、先頭に配置する操作を繰り返すソートアルゴリズムです。

## アルゴリズムの動き
配列を「ソート済み領域」と「未ソート領域」に分け、未ソート領域から最小値を選択してソート済み領域の末尾に追加します。

```
[3, 22, 19, 5] の場合

初期: [] | 3, 22, 19, 5

ステップ1: 未ソート部分の最小値 3 を選択
          [3] | 22, 19, 5

ステップ2: 未ソート部分の最小値 5 を選択
          [3, 5] | 22, 19

ステップ3: 未ソート部分の最小値 19 を選択
          [3, 5, 19] | 22

ステップ4: 残り 22
          [3, 5, 19, 22]
```

## コードの解説

### 主要な処理（main.cpp:12-26）
```cpp
for (size_t i{0}; i < size; ++i)
{
    uint32_t min_idx = i;            // 最小値のインデックスを初期化

    for (size_t j = i + 1; j < size; ++j)  // 未ソート部分を探索
    {
        if (nums[j] < nums[min_idx])
        {
            min_idx = j;             // より小さい値を見つけたら更新
        }
    }

    if (i != min_idx)
        swap(nums[i], nums[min_idx]);  // 最小値を現在位置と交換
}
```

- 外側のループ（`i`）: ソート済み領域の境界
- `min_idx`: 未ソート部分での最小値のインデックス
- 内側のループ（`j`）: 未ソート部分から最小値を探索
- `swap`: 見つけた最小値を現在位置と交換

## 計算量
- **時間計算量**:
  - 最良: O(n²)
  - 平均: O(n²)
  - 最悪: O(n²)
- **空間計算量**: O(1)（追加のメモリ不要）

## 特徴
- **不安定ソート**: 同じ値の順序が保たれない可能性がある
  - 例: `[3a, 3b, 1]` → `[1, 3b, 3a]`（3aと3bの順序が入れ替わる）
- **データに依存しない**: どんなデータでも必ずO(n²)
- **交換回数が少ない**: 最大でもn回の交換しか行わない
- **シンプル**: 理解しやすい

## 使いどころ
- メモリ書き込みが高コストな環境（交換回数が少ないため）
- 小さいデータセット
- アルゴリズムの学習用
- 実用には向かない（より高速なアルゴリズムを使うべき）

## 他のソートとの比較
- **挿入ソート vs 選択ソート**:
  - 挿入ソート: データがほぼソート済みなら高速（最良O(n)）
  - 選択ソート: どんなデータでもO(n²)だが、交換回数が少ない
